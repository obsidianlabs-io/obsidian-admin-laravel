# Obsidian Admin Laravel v1.2.0

A production-hardening release that strengthens architecture discipline, tenant-safe behavior, and CI quality gates without breaking the existing API contract.

## Highlights

- Enforced typed request, action, service, and response boundaries across `Auth`, `Access`, `Tenant`, `Shared`, and `System`
- Unified query actions and shared pagination payloads for high-value list endpoints
- Added architecture guard tests to prevent controller and service boundary regressions
- Hardened auth and session flows for inactive tenants, roles, organizations, and teams
- Added deactivation-first deletion lifecycle for safer delete behavior
- Improved audit logging with log-type classification, filtering, and API access log safeguards
- Added PostgreSQL CI support and fixed workflow reliability issues

## Internal Architecture Improvements

- Replaced remaining controller array contexts with DTOs and typed result objects
- Extracted focused auth/session actions and removed thin token/session indirection
- Typed theme config, audit policy, health, feature flag, CRUD schema, idempotency, tenant context, and OpenAPI inspection boundaries
- Centralized pagination response shaping and list query orchestration

## Quality Gates

- Laravel Pint
- PHPStan / Larastan
- Full feature and architecture test suite
- Architecture boundary assertions for controller and service contracts
