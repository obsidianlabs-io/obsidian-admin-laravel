# Obsidian Admin Laravel v1.3.1

A backend patch release focused on security and contract alignment after the Laravel 13 lane went stable.

## Highlights

- aligned the published auth contract to kebab-case endpoints such as `/auth/refresh-token` and `/auth/user-info`
- tightened system route protection with explicit feature flag and CRUD schema permission middleware
- added missing system permission seeds so the finer-grained middleware model works in fresh and upgraded environments
- fixed security baseline route coverage so release and CI checks understand the new auth route shape
- hardened long-lived worker behavior by flushing request-scoped context after each request
- reduced accidental password reset token exposure by requiring explicit opt-in even in non-production debugging flows
