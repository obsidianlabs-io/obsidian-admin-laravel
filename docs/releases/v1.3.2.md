# Obsidian Admin Laravel v1.3.2

A major refactor release focused on eliminating over-engineering, extracting service boundaries, expanding test coverage, and hardening security.

## Highlights

### Service Extraction & Refactoring
- Extracted `ThemeConfigNormalizer` (444 lines) from `ThemeConfigService` to centralize config validation, defaults, limits, and normalization logic
- Extracted `AuditPolicyResolver` (237 lines) from `AuditPolicyService` to centralize event catalog, policy resolution, scope caching, and normalization
- Refactored `ThemeConfigService` (712 → 277 lines) and `AuditPolicyService` (582 → 364 lines) to delegate to their respective normalizer/resolver classes
- Split the shared `ApiController` into focused controller concerns for JSON responses, authentication helpers, idempotency, optimistic locks, pagination, deletion responses, and trace reporting
- Split monolithic application schema migration into modular schema files (users, cache, jobs, personal tokens, tenants, roles, permissions, audit logs, pulse, etc.)
- Unified audit event lifecycle, aligned `RequestContext`, resolved parameter docblocks, and made broadcasts transaction-safe

### DTO & Result Cleanup
- Deleted 28 thin DTO and Result wrapper files across Auth, FeatureFlag, Language, Role, User, and AuditPolicy domains (net −1,300 lines of pass-through code)
- Inlined `ResolvedUserRoles` as `list<string>`, `SessionRecordsResult` as `list<SessionRecord>`, and `UserProfileSnapshot` as an associative array
- Merged `RevokeSessionActionResult` and `UpdateSessionAliasActionResult` into their respective Service Result classes
- Consolidated five thin feature-flag actions into a single `FeatureFlagAction`

### Security & Performance
- Added a composite database index on `user_id` + `created_at` for both `audit_logs` and `api_access_logs` tables
- Registered `FeatureFlagService` as a singleton to ensure request-scoped override cache is shared across the request lifecycle
- Added an Octane listener `FlushFeatureFlagOverrideCache` to clear request-scoped feature-flag override cache on request end
- Updated `RoleSeeder` to align with current permission model

### Test Coverage
- Added `PermissionApiTest` (7 tests) and `RoleApiTest` (8 tests) for direct route-level authorization coverage
- Added `ThemeConfigNormalizerTest` (20 tests, 70 assertions), `AuditPolicyResolverTest` (14 tests, 342 assertions), and `FeatureFlagServiceTest` (13 tests, 342 assertions)
- Total: 332 passed / 0 failed / 1 skipped

### Bug Fixes
- Added an `admins` password broker alias so environments using `AUTH_PASSWORD_BROKER=admins` can execute forgot-password flows
- Fixed `FeatureFlagService` override cache not sharing across the request lifecycle
- Fixed `UpdateSessionAliasResult.toArray()` missing `sessionId` in its output payload
- Kept the role store input DTO compatible with existing HTTP aliases
