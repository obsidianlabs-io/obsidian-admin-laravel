# Obsidian Admin Laravel v1.3.2

A backend maintenance patch focused on API controller maintainability, role DTO compatibility, and release-test stability.

## Highlights

- Split the shared `ApiController` into focused controller concerns for response formatting, authentication helpers, idempotency, optimistic locks, pagination, deletion responses, and trace reporting
- Reduced repeated RoleController authentication, context, and authorization setup while keeping the published role API contract stable
- Kept role store input compatibility for existing HTTP aliases while aligning the input DTO with the domain role creation DTO
- Added an `admins` password broker alias for environments configured with `AUTH_PASSWORD_BROKER=admins`
- Documented the migration policy for future schema changes so new changes land in independent timestamped migration files
