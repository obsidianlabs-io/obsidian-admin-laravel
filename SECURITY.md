# Security Policy

## Supported Versions

Security fixes are provided for:

- the latest release line
- the current `main` branch, when a fix has not yet been released

Older versions may not receive patches.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report vulnerabilities privately using one of these channels:

- GitHub Security Advisories, if enabled for the repository
- Email: `obsidianlabs.tech@gmail.com`

Please include:

- a short summary of the issue
- affected endpoint, module, or workflow
- clear reproduction steps
- impact assessment
- any proof-of-concept details that help verify the report

If the issue involves tenant isolation, authentication, authorization, audit integrity, or token/session handling, say that explicitly.

## Response Expectations

We aim to:

- acknowledge valid reports promptly
- reproduce and assess severity
- prepare a fix or mitigation
- publish a release once the fix is ready

Please avoid disclosing the issue publicly until a fix or mitigation is available.
